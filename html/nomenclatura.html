Buenas prácticas en nomenclatura de clases

¿Qué es?

Las buenas prácticas en nomenclatura de clases son formas recomendables para nombrar clases permitiendo tener un código limpio, entendible diguiendo un patrón para el lector e incluso brindando información sobre la función de la clase.


Convenciones útiles

Camel case

Se caracteriza por tener la primera letra en minúscula y cada palabra siguiente empezando en mayúscula:

mensajeDeError
obtenerResultado
edadDelUsuario

Es usada en #hashTags, variables, constantes, funciones y métodos en varios lenguajes de programación como PHP, Java, JavaScript.

Pascal case

También conocido como "upper camel case" o "capital case", se caracteriza por tener todas las primeras letras de cada palabra en mayúscula:

MensajeDeError
ObtenerResultado
EdadDelUsuario

Es recomendable usarla para clases ya sea para Java, JavaScript, etc.

Snake case

También conocido como "underscore case", se caracteriza por poner guión bajo para separar las palabras, cuando las palabras están en mayúscula, se le conoce como "screaming snake case":

mensaje_de_error
obtener_resultado
EDAD_DEL_USUARIO

Es recomendable usar SCREAMING_SNAKE_CASE para constantes.

Kebab case

Se caracteriza por el uso del guión seguido de cada palabra, cuando las palabras están en mayúsculas, se le llama "screaming kebab case":

mensaje-de-error
obtener-resultado
EDAD-DEL-USUARIO

Se puede usar en nombres de clases y IDs como CSS, en nombres de archivos y URLs.


Ejemplo:

  • camelCase: Variables, constantes, funciones y metodos.
  • PascalCase: Para clases.

    • Evitar colisiones y mantener la claridad del código

    Usar nombres claros y descriptivos para las variables en CSS y JS ayuda a que el código sea más ordenado, fácil de entender y mantener. Además, seguir una convención adecuada mejora el trabajo en equipo, especialmente en proyectos grandes.

    Namespacing: Ayuda a evitar colisiones de nombres al prefijar las variables con un identificador como system- o project-, pudiendo identificar claramente el origen y evitando conflictos en el código.
